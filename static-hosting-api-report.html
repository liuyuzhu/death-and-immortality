静态网站托管平台调研报告

对比主流静态托管平台的 API 能力、免费额度、部署方式、易用性，帮助你选择最合适的方案

📅 2026年4月30日

📋 调研范围和方法

本次调研覆盖以下主流支持 API 部署的静态网站托管平台：

🏆 最终推荐排名：Netlify > GitHub Pages > Cloudflare Pages > Vercel > 其他

评判标准：API 完备程度、免费额度、部署便捷性、全球 CDN、自定义域名、国内访问速度。

🔍 各平台详细分析

🥇 1. Netlify 免费版最简单

REST API 最完善，上传 ZIP 包即可部署，最适合程序化部署场景

属性说明
网站netlify.com
API 方式REST API（OAuth 2.0 认证） + CLI (netlify-cli)
免费额度100GB 带宽/月、300 构建分钟/月、无限站点
上传方式上传 ZIP 压缩包 → 自动解压部署
自定义域名✅ 免费支持 + 自动 SSL
CDN✅ 全球 CDN (Fastly)
国内访问⚠️ 一般

📌 API 示例（cURL）

# 1. 创建站点（首次）
curl -H "Content-Type: application/json" \
  -H "Authorization: Bearer $NETLIFY_TOKEN" \
  -d '{"name":"my-site"}' \
  https://api.netlify.com/api/v1/sites

# 2. 上传 ZIP 包部署
curl -H "Content-Type: application/zip" \
  -H "Authorization: Bearer $NETLIFY_TOKEN" \
  --data-binary "@site.zip" \
  https://api.netlify.com/api/v1/sites/$SITE_ID/deploys

📌 Python 示例

import requests

# 部署 ZIP 包
headers = {
  "Authorization": f"Bearer {token}"
}
with open("site.zip", "rb") as f:
  resp = requests.post(
    f"https://api.netlify.com/api/v1/sites/{site_id}/deploys",
    headers=headers, data=f
  )
print(resp.json()["ssl_url"]) # 输出部署后的 HTTPS URL

🥈 2. Cloudflare Pages 免费版需 Wrangler CLI

无限带宽，全球最快的边缘网络，中国访问速度最好的国际平台

属性说明
网站pages.cloudflare.com
API 方式REST API + Wrangler CLI（推荐）+ Direct Upload API
免费额度无限带宽、无限请求、500 构建/月
上传方式Direct Upload API（上传 ZIP）或 Wrangler CLI 部署
自定义域名✅ 免费支持 + 自动 SSL
CDN✅ 全球 CDN（330+ 城市节点，含中国节点）
国内访问✅ 最佳（有国内节点）

📌 API 示例（cURL）

# 1. 上传 ZIP 部署
curl -X POST \
  "https://api.cloudflare.com/client/v4/accounts/$ACCOUNT_ID/pages/projects/$PROJECT/deployments" \
  -H "Authorization: Bearer $CF_TOKEN" \
  -F "file=@site.zip"

# 2. 或使用 Wrangler CLI（推荐）
wrangler pages deploy ./public --project-name my-site

📌 Python 示例

# 使用 Cloudflare Pages Direct Upload API
headers = {
  "Authorization": f"Bearer {cf_token}"
}
files = {"file": open("site.zip", "rb")}
resp = requests.post(
  f"https://api.cloudflare.com/client/v4/accounts/{account_id}/pages/projects/{project}/deployments",
  headers=headers, files=files
)
print(resp.json()["result"]["url"])

🥉 3. GitHub Pages 完全免费

无带宽限制，通过 GitHub API 推送文件即可，虽然步骤略多但最稳定

属性说明
网站pages.github.com
API 方式GitHub REST API（创建/更新仓库文件）
免费额度无限带宽、无限存储（单文件 ≤100MB）
上传方式通过 GitHub API 创建/更新仓库文件 → 自动触发 Pages 构建
自定义域名✅ 免费支持 + 自动 SSL（需 DNS 配置）
CDN✅ Fastly CDN
国内访问⚠️ 较慢

📌 API 示例（cURL）

# 1. Base64 编码 HTML 文件
base64 -w0 index.html > index.html.b64

# 2. 创建/更新文件（用 GitHub API）
curl -X PUT \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Update via API",
    "content": "'$(cat index.html.b64)'",
    "branch": "main"
  }' \
  https://api.github.com/repos/$OWNER/$REPO/contents/index.html

# 3. Pages 会自动构建并部署

📌 Python 示例

import base64, requests

# Base64 编码 HTML
with open("index.html", "rb") as f:
  b64 = base64.b64encode(f.read()).decode()

# 通过 GitHub API 推送
data = {
  "message": "Update via API",
  "content": b64,
  "branch": "main"
}
resp = requests.put(
  f"https://api.github.com/repos/{owner}/{repo}/contents/index.html",
  headers={"Authorization": f"Bearer {token}"},
  json=data
)

🏆 最终推荐

🥇 最佳选择

Netlify

适用场景：需要程序化自动部署 HTML，且可以接受国内访问速度一般的场景。

理由：REST API 最成熟完整，直接上传 ZIP 包就可以，不需要安装任何工具，全世界广泛使用。

API 方式：POST ZIP → /api/v1/sites/{id}/deploys

🥈 国内访问最佳

Cloudflare Pages

适用场景：需要国内访问速度快、无限带宽、且愿意使用 Wrangler CLI。

理由：无限带宽，国内有 CDN 节点，Direct Upload API 也支持 ZIP 上传。

API 方式：POST ZIP → /pages/projects/{name}/deployments

🥉 已有 GitHub 生态

GitHub Pages

适用场景：你已经在使用 GitHub，想把 HTML 放在已有仓库中。

理由：完全免费，无限带宽，可以通过 GitHub API 直接写入文件。虽然步骤稍多（Base64 编码 + 推送），但最稳定可靠。

注意：GitHub Pages 不支持直接 API 部署，需要通过 Content API 创建/更新文件来触发自动构建。

🚀 Python 一键部署脚本

以下脚本可以让你用几行代码将任意 HTML 文件部署到任意平台：

#!/usr/bin/env python3
# deploy_html.py — 通过 API 上传 HTML 到多个平台

import requests
import base64, zipfile, io, json


def deploy_netlify(html_file, site_id, token):
    """上传 HTML 到 Netlify"""
    headers = {"Authorization": f"Bearer {token}"}
    resp = requests.post(
        f"https://api.netlify.com/api/v1/sites/{site_id}/deploys",
        headers=headers,
        files={"file": open(html_file, "rb")}
    )
    return resp.json()["ssl_url"]


def deploy_github_pages(html_file, owner, repo, branch, token):
    """推送 HTML 到 GitHub Pages 仓库"""
    with open(html_file, "rb") as f:
        b64 = base64.b64encode(f.read()).decode()
    data = {
        "message": "Deploy via API",
        "content": b64,
        "branch": branch
    }
    resp = requests.put(
        f"https://api.github.com/repos/{owner}/{repo}/contents/index.html",
        headers={"Authorization": f"Bearer {token}"},
        json=data
    )
    return f"https://{owner}.github.io/{repo}/"

✅ 总结

需求场景推荐平台理由
📄 想用最少的代码上传 HTML Netlify API 最简洁，上传 ZIP 即可，5 行 Python 搞定
🇨🇳 需要国内用户访问快 Cloudflare Pages 唯一有国内 CDN 节点的国际平台，无限带宽
🐙 已经在用 GitHub 管理代码 GitHub Pages 完全免费，无限带宽，与 GitHub 生态无缝集成
⚡ 想用最少的配置 Vercel Deploy Hooks 1 行 cURL 就能触发部署
💰 预算有限但要最好 Cloudflare Pages 无限带宽完全免费，没有使用限制
